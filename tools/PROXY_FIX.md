# 代理端口问题解决方案

## 问题说明

您当前的系统代理是 `127.0.0.1:9000`，但我们的工具默认使用 `8080` 端口。

## 解决方案

### 方案一：使用新的启动器（推荐）

运行新的启动器，它支持 9000 端口：

```bash
python network_monitor_launcher_v2.py
```

然后选择：
- **选项 2** - mitmweb + 自动设置系统代理（端口9000）

### 方案二：使用原启动器 + 手动配置端口

如果您想继续使用原启动器，但需要用 9000 端口：

1. 不使用自动代理功能（选择选项 2）
2. 手动配置浏览器代理为 127.0.0.1:9000
3. 启动 mitmproxy 时指定端口：

```bash
mitmweb -p 9000 -s network_monitor_v2.py
```

### 方案三：直接使用应急恢复工具

如果您现在无法上网，请先运行：

```bash
python restore_proxy.py
```

## 新的启动器功能

`network_monitor_launcher_v2.py` 提供：
- 选项 1 - 使用 8080 端口（默认）
- 选项 2 - 使用 9000 端口（匹配您当前设置）
- 选项 3 - 自定义端口
- 其他功能与原版相同

## 注意事项

- 如果程序异常退出，请运行 `python restore_proxy.py` 恢复代理
- 确保 mitmproxy 正在运行时再使用自动代理功能
- 按 Ctrl+C 正常退出会自动恢复代理
